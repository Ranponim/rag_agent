# LangGraph RAG Agent 학습용 샘플 문서
# =====================================
# 이 파일은 RAG 예제에서 사용할 샘플 문서입니다.
# 실제 프로젝트에서는 다양한 형식의 문서 (PDF, CSV, JSON 등)를 사용할 수 있습니다.

## LangGraph 소개

LangGraph는 LangChain 팀에서 개발한 라이브러리로, 대규모 언어 모델(LLM)을 사용하여 
상태를 가진(stateful) 다중 행위자(multi-actor) 애플리케이션을 구축하기 위한 도구입니다.

LangGraph의 주요 특징:
1. 상태 관리: 그래프의 각 노드가 공유 상태를 읽고 쓸 수 있습니다.
2. 사이클 지원: 일반적인 DAG와 달리 순환 구조를 지원합니다.
3. 제어 가능: 에이전트의 실행 흐름을 세밀하게 제어할 수 있습니다.
4. 영속성: 체크포인트를 통해 상태를 저장하고 복원할 수 있습니다.

## RAG (Retrieval-Augmented Generation)

RAG는 검색 증강 생성의 약자로, 외부 지식 소스에서 관련 정보를 검색하여 
LLM의 응답 품질을 향상시키는 기법입니다.

RAG의 기본 구성 요소:
1. 문서 로더 (Document Loader): 다양한 형식의 문서를 로드합니다.
2. 텍스트 분할기 (Text Splitter): 긴 문서를 청크(chunk)로 나눕니다.
3. 임베딩 모델 (Embedding Model): 텍스트를 벡터로 변환합니다.
4. 벡터 저장소 (Vector Store): 임베딩 벡터를 저장하고 검색합니다.
5. 검색기 (Retriever): 쿼리와 유사한 문서를 검색합니다.

## LangGraph의 핵심 개념

### State (상태)
상태는 그래프를 통해 전달되는 공유 데이터 구조입니다.
TypedDict를 사용하여 상태 스키마를 정의합니다.

### Node (노드)
노드는 그래프에서 작업을 수행하는 단위입니다.
각 노드는 현재 상태를 입력 받아 업데이트된 상태를 반환합니다.

### Edge (엣지)
엣지는 노드 간의 연결을 정의합니다.
단순 엣지는 다음에 실행할 노드를 직접 지정합니다.
조건부 엣지는 조건에 따라 다른 노드로 분기할 수 있습니다.

### Compile (컴파일)
StateGraph를 정의한 후 compile() 메서드를 호출하여 실행 가능한 그래프로 변환합니다.
컴파일된 그래프는 invoke() 또는 stream() 메서드로 실행할 수 있습니다.

## Entity RAG

Entity RAG는 일반적인 RAG를 개선한 방식으로, 문서에서 엔티티(개체)를 추출하고
이를 기반으로 더 정확한 검색을 수행합니다.

엔티티 유형:
- 인물 (Person): 사람 이름
- 조직 (Organization): 회사, 기관 등
- 위치 (Location): 장소, 지역 등
- 날짜 (Date): 시간 정보
- 기술 (Technology): 기술 스택, 도구 등

## Advanced RAG 기법

### Self-RAG
Self-RAG는 LLM이 스스로 검색이 필요한지 판단하고,
검색된 문서의 관련성과 생성된 응답의 품질을 평가하는 기법입니다.

### Corrective RAG
Corrective RAG는 검색된 문서의 관련성을 평가하고,
관련 없는 문서는 제외하거나 웹 검색으로 보완하는 기법입니다.

### Adaptive RAG
Adaptive RAG는 쿼리의 복잡도에 따라 적절한 검색 전략을 선택합니다.
간단한 질문은 직접 응답하고, 복잡한 질문은 다단계 검색을 수행합니다.
